## Design Specifications

### Modal Dimensions
- **Width:** 100% with 20px padding (max 400px)
- **Border Radius:** 20px
- **Padding:** 24px
- **Background:** White (#ffffff)

### Icon Container
- **Size:** 96x96px
- **Border Radius:** 48px (circular)
- **Margin Bottom:** 20px

### Typography
- **Title:** 
  - Font size: 22px (h2)
  - Font weight: 700 (bold)
  - Color: #1e293b (dark slate)
  - Margin bottom: 12px

- **Message:**
  - Font size: 15px (body)
  - Font weight: 400 (regular)
  - Color: #64748b (slate)
  - Line height: 22px
  - Margin bottom: 24px

- **Button Text:**
  - Font size: 16px
  - Font weight: 600 (semibold)
  - Color: White

### Buttons
- **Primary Button:**
  - Background: Primary green (Colors.primary)
  - Padding: 14px vertical, 32px horizontal
  - Border radius: 12px
  - Shadow: 0px 2px 4px rgba(primary, 0.2)

- **Secondary Button (Skip):**
  - Background: White
  - Border: 1px solid #e2e8f0
  - Padding: 14px vertical
  - Border radius: 12px
  - Text color: #64748b

### Progress Indicators (Tour)
- **Inactive Dot:**
  - Size: 8x8px
  - Border radius: 4px
  - Background: #e2e8f0 (light gray)

- **Active Dot:**
  - Width: 24px
  - Height: 8px
  - Border radius: 4px
  - Background: Primary green

- **Spacing:** 8px gap between dots

### Overlay
- **Background:** rgba(0, 0, 0, 0.5)
- **Blur:** Optional backdrop blur
- **Tap to dismiss:** No (must use button)

### Animations
- **Modal Entry:** Fade in (200ms)
- **Modal Exit:** Fade out (150ms)
- **Button Press:** Scale down slightly (0.95)

## Accessibility Features

### Screen Reader Support
- All modals have descriptive labels
- Icons have text alternatives
- Buttons have clear action descriptions

### Touch Targets
- All buttons: Minimum 44x44pt
- Close button: 32x32pt (larger than minimum)
- Adequate spacing between interactive elements

### Color Contrast
- Text on white: WCAG AA compliant
- Button text on green: High contrast
- Icon colors: Clearly visible

### Keyboard Navigation
- Can dismiss with back button
- Focus management for accessibility
- Logical tab order

## Responsive Behavior

### Small Screens (< 360px)
- Modal width: 90% of screen
- Reduced padding: 20px
- Smaller icon: 80x80px
- Font sizes scale down slightly

### Large Screens (> 600px)
- Modal max width: 400px
- Centered on screen
- Maintains aspect ratio

### Landscape Orientation
- Modal scrollable if needed
- Maintains readability
- Buttons remain accessible

## Animation Timing

```
Modal Appearance:
├─ Overlay fade in: 200ms
├─ Modal scale up: 200ms (from 0.9 to 1.0)
└─ Content fade in: 150ms (delayed 50ms)

Modal Dismissal:
├─ Content fade out: 100ms
├─ Modal scale down: 150ms (to 0.95)
└─ Overlay fade out: 150ms

Tour Step Transition:
├─ Current step fade out: 100ms
├─ Progress dot animation: 150ms
└─ Next step fade in: 150ms (delayed 50ms)
```

## Usage Context

### When to Show Each Modal

1. **Empty Favorites:** First visit with no items
2. **Sold Items:** When ≥1 favorite is sold
3. **Price Drop:** When ≥1 favorite has price reduction
4. **First Notification:** On receiving first notification
5. **Types Tour:** First visit to notifications screen
6. **Unread Reminder:** After 48 hours of unread notifications

### Priority Order (if multiple triggers)
1. First notification welcome
2. Notification types tour
3. Empty favorites
4. Sold items notification
5. Price drop alert
6. Unread reminder
