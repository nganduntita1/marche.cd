## User Interaction Flow

### Flow 1: First-Time Seller

```
User Opens Dashboard
        ↓
Dashboard Tour Appears
        ↓
User Reads Features
        ↓
User Clicks "Got it!"
        ↓
Tour Dismissed
        ↓
Tour Won't Show Again
```

### Flow 2: Low-View Listing

```
System Detects Low Views
        ↓
Warning Card Appears
        ↓
User Sees Suggestions
        ↓
User Chooses Action
   ↙         ↘
Edit       Promote
   ↓           ↓
Navigate   Navigate
to Edit    to Listing
```

### Flow 3: Quick Response

```
User Clicks "Quick Responses"
        ↓
Templates Modal Opens
        ↓
User Browses Templates
        ↓
User Selects Template
        ↓
Template Text Inserted
        ↓
User Customizes (Optional)
        ↓
User Sends Message
```

### Flow 4: Mark as Sold

```
User Initiates Mark as Sold
        ↓
Guidance Modal Appears
        ↓
User Reads Process
        ↓
User Clicks "Continue"
        ↓
Select Buyer Screen
        ↓
Rate Transaction
        ↓
Complete Sale
```

### Flow 5: Learn About Promotions

```
User Clicks "Promotions"
        ↓
Options Modal Opens
        ↓
User Reviews 3 Options
        ↓
User Compares Costs/Benefits
        ↓
User Clicks "Got it!"
        ↓
User Makes Decision
```

## Color Coding

### Status Colors
- 🟢 **Green** - Success, Active, Good performance
- 🟡 **Yellow** - Warning, Needs attention
- 🔴 **Red** - Error, Critical issue
- 🔵 **Blue** - Information, Neutral
- 🟣 **Purple** - Premium, Promotion

### Component Colors
- **Primary** - #22c55e (Green)
- **Warning** - #f59e0b (Orange)
- **Error** - #ef4444 (Red)
- **Info** - #3b82f6 (Blue)
- **Background** - #f8fafc (Light Gray)
- **Text** - #1e293b (Dark Gray)

## Responsive Behavior

### Mobile (< 768px)
- Single column layout
- Full-width cards
- Stacked quick actions
- Bottom sheet modals

### Tablet (768px - 1024px)
- Two column layout
- Side-by-side quick actions
- Centered modals
- Optimized spacing

### Desktop (> 1024px)
- Multi-column layout
- Inline quick actions
- Larger modals
- Enhanced spacing

## Accessibility

### Touch Targets
- Minimum 44x44pt for all buttons
- Adequate spacing between elements
- Clear visual feedback on press

### Screen Readers
- All icons have text labels
- Proper heading hierarchy
- Descriptive button labels
- Modal announcements

### Keyboard Navigation
- Tab order follows visual flow
- Enter/Space activates buttons
- Escape closes modals
- Focus indicators visible

## Animation Timing

### Modal Transitions
- Fade in: 300ms
- Slide up: 400ms
- Fade out: 200ms

### Card Animations
- Appear: 400ms spring
- Dismiss: 300ms ease-out
- Hover: 150ms ease-in-out

### Button Feedback
- Press: 100ms
- Release: 150ms
- Ripple: 600ms

## State Indicators

### Loading States
- Skeleton screens for data
- Spinner for actions
- Progress bars for uploads

### Empty States
- Friendly illustrations
- Helpful messages
- Clear call-to-action

### Error States
- Clear error messages
- Suggested solutions
- Retry options

## Best Practices

### Do's ✅
- Show tour on first visit
- Provide actionable suggestions
- Use clear, simple language
- Offer quick actions
- Support both languages

### Don'ts ❌
- Don't show tour repeatedly
- Don't overwhelm with info
- Don't use technical jargon
- Don't hide important actions
- Don't forget translations
